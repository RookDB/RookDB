Decode a serialized row back into typed values (logical order). total_row_size must be the byte length of row_bytes (available from the slot directory); it is used to derive the last var-len payload length.
row_byte_size
Returns the exact on-disk byte size of a row given a schema and concrete values, without constructing a Row or serialising anything. Useful for pre-insert capacity checks in the heap/page manager.
serialize_nullable_row
Encode a row from string literals. Layout: [Header 4B][Null Bitmap][Var-Len Offset Table][Fixed Data][Var-Len Data] Var-len payloads are stored without a length prefix; lengths are derived from adjacent offset-table entries and the total row size.
