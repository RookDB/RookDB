Physical schema and row layout computation for the new tuple format.

The logical schema is the user-defined column order (as in CREATE TABLE). The physical schema regroups columns so all fixed-length columns come first, followed by all variable-length columns. This enables O(1) access to any fixed-length column without scanning variable-length data.

On-disk row format:

[Header 4B] [Null Bitmap] [Var-Len Offset Table] [Fixed-Length Data] [Var-Len Data]

Structs§

PhysicalSchema
Reorders logical columns into two physical groups: Group 1 (fixed): all fixed-length columns, in their original logical order. Group 2 (var): all variable-length columns, in their original logical order.
RowLayout
Byte-level geometry of a row, derived from a PhysicalSchema.
