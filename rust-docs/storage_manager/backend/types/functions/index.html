Built-in SQL scalar functions operating on DataValue.

All functions follow these conventions:

  • They accept typed DataValue references (not raw strings).
  • They return FunctionError::TypeMismatch if the input type is not supported by that function.
  • String functions (upper, lower, trim, etc.) always return DataValue::Varchar even when given a CHAR input, matching SQL standard behaviour.
